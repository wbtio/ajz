# خطة بناء تطبيق JAZ للعميل

> تطبيق موبايل موجّه للعميل (iOS + Android) يرافقه من اكتشاف الفعالية الدولية إلى عودته من السفر، متصل بنفس قاعدة بيانات الموقع.

---

## 📌 ملخص تنفيذي

| البند | القرار |
|---|---|
| **الإطار (Framework)** | React Native + **Expo** |
| **اللغة** | TypeScript (نفس لغة الموقع) |
| **قاعدة البيانات / الخلفية** | **Supabase** (نفس قاعدة بيانات الموقع — لا قاعدة جديدة) |
| **المصادقة** | Supabase Auth (نفس حسابات الموقع) |
| **المنصات** | iOS + Android من كود واحد |
| **اللغة المعروضة** | عربي (RTL) + إنجليزي |

**لماذا هذا الإطار؟** يعيد استخدام مهارات فريق الويب وكود Supabase، ويتشارك نفس قاعدة البيانات مع الموقع — وهذا ما يحقّق ميزة «يبدأ التسجيل بالموقع ويُكمله بالتطبيق».

---

## 🧰 الحزم الأساسية (Libraries)

| الغرض | الحزمة |
|---|---|
| التنقّل بين الشاشات | `expo-router` |
| الاتصال بقاعدة البيانات | `@supabase/supabase-js` |
| الإشعارات الفورية | `expo-notifications` |
| الكاميرا ورفع الوثائق | `expo-camera` + `expo-image-picker` |
| بطاقة QR | `react-native-qrcode-svg` |
| التخزين المحلي (أوفلاين) | `@react-native-async-storage/async-storage` |
| إدارة الحالة والبيانات | `@tanstack/react-query` |
| الخطوط العربية | `expo-font` (IBM Plex Sans Arabic) |

## المرحلة 0 — التحضير والإعداد

**الهدف:** بيئة عمل جاهزة وهوية بصرية موحّدة قبل بناء أي شاشة.

- [ ] إنشاء مشروع Expo + TypeScript + `expo-router`
- [ ] ربط مفاتيح Supabase (نفس مشروع الموقع)
- [ ] إعداد دعم RTL والخطوط (IBM Plex Sans Arabic)
- [ ] بناء «نظام التصميم»: الألوان (الأحمر `#8b0000`)، الأزرار، البطاقات، الحقول
- [ ] إعداد نظام الترجمة (عربي/إنجليزي) المشترك مع الموقع
- [ ] هيكلة المجلدات والتنقّل الأساسي (Tabs)

**المخرج:** تطبيق فارغ يفتح بشاشة بيضاء بالهوية الصحيحة ويتصل بقاعدة البيانات.

## المرحلة 1 — الأساس (الدخول + الرئيسية)

**الهدف:** دخول العميل بحسابه ووصوله لشاشة رئيسية حيّة.

- [ ] **شاشة الدخول** عبر Supabase Auth (نفس حساب الموقع)
- [ ] شريط **«أكمل من حيث توقفت»** — قراءة التسجيلات غير المكتملة من جدول `registrations`
- [ ] **الشاشة الرئيسية:** عدّاد الفعالية القادمة + أزرار سريعة + توصيات حسب القطاع
- [ ] شريط التنقّل السفلي (الرئيسية · الفعاليات · تذكرتي · حسابي)

**يعتمد على:** المرحلة 0.
**المخرج:** العميل يدخل ويشوف بياناته الحقيقية من قاعدة البيانات.

---

## المرحلة 2 — الفعاليات والتفاصيل

**الهدف:** اكتشاف الفعاليات الدولية واتخاذ قرار المشاركة.

- [ ] **شاشة الفعاليات:** قائمة من جدول `events` مع بحث وفلترة (قطاع · دولية/محلية · مميزة · تاريخ)
- [ ] **شاشة تفاصيل الفعالية:** كل المعلومات + شارات + زر تسجيل ثابت
- [ ] **حفظ المفضّلة** (يتطلب جدول جديد `saved_events` أو حقل مفضّلات)
- [ ] **مشاركة كصورة** — توليد بطاقة الفعالية ومشاركتها

**يعتمد على:** المرحلة 1.
**المخرج:** العميل يتصفّح ويصل لأي فعالية ويقرّر التسجيل.

---

## المرحلة 3 — التسجيل والمزامنة (الأهم)

**الهدف:** تحقيق ميزة «يبدأ بالموقع ويُكمل بالتطبيق».

- [ ] **نموذج تسجيل متعدّد الخطوات** مع مؤشّر تقدّم
- [ ] **الحفظ التلقائي** في `registrations.form_data` بعد كل خطوة (لا يضيع شيء عند الانقطاع)
- [ ] **التعبئة المسبقة** من بيانات حساب العميل
- [ ] **وثائق خاصة بكل فعالية** (تُقرأ من `registration_config` في جدول `events`)
- [ ] **شعار داخل الموقع**: «أكمل خطواتك من التطبيق» (تعديل بسيط على الموقع)

**يعتمد على:** المرحلة 2.
**المخرج:** تسجيل متصل بالكامل بين الموقع والتطبيق.

## المرحلة 4 — دعم الدعوات والتأشيرات (الميزة المميّزة)

**الهدف:** نقل خدمة خطاب الدعوة ودعم التأشيرة إلى الهاتف.

- [ ] **شاشة الطلب** + رفع الوثائق عبر **الكاميرا** (`expo-camera`) إلى Supabase Storage
- [ ] **قائمة الوثائق المطلوبة** (تختلف حسب الفعالية)
- [ ] **تتبّع 4 مراحل:** تقديم ← تدقيق ← إصدار الخطاب ← تسهيل السفر
- [ ] **إشعار** عند كل تغيير في الحالة
- [ ] *(يتطلب جدول جديد:* `invitation_requests` *+ جدول وثائق مرتبط)*

**يعتمد على:** المرحلة 3.
**المخرج:** العميل يقدّم طلب دعوته ويتابعه بالكامل من الهاتف.

- [ ] *(يتطلب جدول جديد:* `trip_itineraries` *)*

**يعتمد على:** المرحلة 4.

## ♻️ أصول وملفات جاهزة يمكن إعادة استخدامها

ميزة كبيرة: المطوّر **لا يبدأ من الصفر**. كثير من ملفات الموقع تُنقل للتطبيق كما هي أو بتعديل بسيط، وبعضها يفيد في أكثر من مهمة:

### الشعارات والصور (مجلد `public/`)
- **الشعار الرئيسي:** `Joint Annual Zone logo.png` — للأيقونة وشاشة البداية والهيدر.
- **شعارات القطاعات الأربعة (SVG):** `JAZ Medical.svg` · `JAZ Technology.svg` · `JAZ Industrie.svg` · `JAZ Academia.svg` — تُستخدم في الفلترة، بطاقات القطاعات، والتوصيات.
- **خلفيات وصور جاهزة:** `hero-globe-bg.png` · `invitation-hero-bg.png` · `iraq-map.png` · `world-map.svg` · `og-image.png` — تصلح للخلفيات وبطاقة المشاركة.

### الكود المنطقي (مجلد `lib/`) — يُنقل مباشرة
- **`database.types.ts`** — أنواع TypeScript لكل جداول قاعدة البيانات؛ تُنسخ كما هي = توفير وقت كبير وأمان للبيانات.
- **`i18n/translations.ts`** — كل النصوص العربية/الإنجليزية جاهزة؛ لا حاجة لإعادة الترجمة.
- **`chatbot/faq.ts`** — أسئلة وأجوبة المساعد الذكي جاهزة للمرحلة 6.
- **`form-fields.ts`** — تعريفات حقول النماذج؛ يُعاد استخدامها في شاشة التسجيل.
- **`supabase/client.ts`** — طريقة الاتصال بقاعدة البيانات (بتعديل بسيط لبيئة الموبايل).
- **`events-visibility.ts`** + **`utils.ts`** — منطق عرض الفعاليات ودوال مساعدة.

### بيانات جاهزة (نفس قاعدة Supabase)
- آلاف الفعاليات، القطاعات، المقالات، والبيانات الثنائية اللغة — **متاحة فوراً** للتطبيق دون أي إدخال جديد.

> **الخلاصة:** الشعارات والأنواع والترجمات والـ FAQ والبيانات كلها قابلة لإعادة الاستخدام، مما يختصر زمن التطوير بشكل ملحوظ — خاصة في المراحل 1، 2، 3، و6.

- **كل مرحلة قابلة للإطلاق** بشكل مستقل (إصدار تجريبي بعد كل مرحلة).
