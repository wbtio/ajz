# خطة بناء تطبيق JAZ للعميل

> تطبيق موبايل موجّه للعميل (iOS + Android) يرافقه من اكتشاف الفعالية الدولية إلى عودته من السفر، متصل بنفس قاعدة بيانات الموقع.
>
> **ملفات مترابطة:** هذا الملف (الجدول الزمني) ↔ [توثيق-تطبيق-JAZ.md](توثيق-تطبيق-JAZ.md) (توثيق التطبيق الكامل) ↔ [توثيق-الموقع-ولوحة-التحكم-JAZ.md](توثيق-الموقع-ولوحة-التحكم-JAZ.md) (الموقع + لوحة التحكم).

---

## 🚦 حالة التنفيذ (آخر تحديث: 1 يوليو 2026)

| المرحلة | الحالة | ملاحظات |
|---|---|---|
| **0** التحضير والإعداد | ✅ مكتملة | المشروع شغّال ومتصل بقاعدة البيانات |
| **1** الأساس (الدخول + الرئيسية + الملف الشخصي) | 🟡 جزئياً | الدخول/الإنشاء + الملف الشخصي الإلزامي (5 أسئلة، تطبيق وموقع) مكتملة؛ **OTP لم يُبنَ بعد** |
| **2** الفعاليات والتفاصيل | 🟡 جزئياً | الشاشات الأساسية جاهزة (بحث/فلترة/تفاصيل)؛ المفضّلة والمشاركة كصورة لاحقاً |
| **3 → 6** | ⬜ لم تبدأ | المرحلة 3 دُمجت (تسجيل + دعم الدعوات في مرحلة واحدة، انظر التصحيح أدناه) |

**ما تم تنفيذه فعلياً (داخل مجلد `mobile/`):**
- مشروع Expo + TypeScript + expo-router، RTL، وخط IBM Plex Sans Arabic.
- نظام تصميم بالهوية الحمراء `#8b0000` ([theme.ts](mobile/src/lib/theme.ts)).
- عميل Supabase للموبايل متصل بنفس قاعدة بيانات الموقع ([supabase.ts](mobile/src/lib/supabase.ts)).
- **مصادقة كاملة:** تسجيل دخول و**إنشاء حساب داخل التطبيق** (مطابق لتسجيل الموقع: الاسم + الهاتف + البريد + كلمة المرور) عبر Supabase Auth ([auth.tsx](mobile/src/lib/auth.tsx)). *(دخول Google كان مضافاً ثم حُذف بطلب صريح — البريد وكلمة المرور فقط حالياً)*
- شاشات الدخول/الإنشاء **بنفس تصميم صفحات الموقع** (أيقونات داخل الحقول، صندوق خطأ أحمر، عنوان فرعي) + الشعار الحقيقي.
- **شاشة الملف الشخصي — إلزامية، 5 أسئلة، بلا تخطّي** بعد إنشاء الحساب: التخصص الرئيسي (صناديق) + التخصص الفرعي (نص حر) + اسم الشركة + المدينة + الدولة ([profile-setup.tsx](mobile/src/app/profile-setup.tsx)). نفس الشاشة أُضيفت **للموقع أيضاً** بعد التسجيل/الدخول إذا كان الملف ناقصاً ([app/auth/complete-profile/page.tsx](app/auth/complete-profile/page.tsx)).
- شريط تبويبات (الرئيسية · الفعاليات · تذكرتي · حسابي) + شاشات الفعاليات والتفاصيل تقرأ بيانات حقيقية من جدول `events`.

**تعديل على الموقع (web):** أُضيف زرّا **«دخول» و«تسجيل»** في هيدر الموقع (للديسكتوب والموبايل) لأنهما كانا غير ظاهرين، فلم يكن المستخدم يصل لصفحة التسجيل ([components/layout/header.tsx](components/layout/header.tsx)).

## 📌 ملخص تنفيذي

| البند | القرار |
|---|---|
| **الإطار (Framework)** | React Native + **Expo** |
| **اللغة** | TypeScript (نفس لغة الموقع) |
| **قاعدة البيانات / الخلفية** | **Supabase** (نفس قاعدة بيانات الموقع — لا قاعدة جديدة) |
| **المصادقة** | Supabase Auth (نفس حسابات الموقع) |
| **المنصات** | iOS + Android من كود واحد |
| **اللغة المعروضة** | عربي (RTL) + إنجليزي |

**لماذا هذا الإطار؟** يعيد استخدام مهارات فريق الويب وكود Supabase، ويتشارك نفس قاعدة البيانات مع الموقع — وهذا ما يحقّق ميزة «يبدأ التسجيل بالموقع ويُكمله بالتطبيق».

## المرحلة 3 — التسجيل الشامل (خدمات مدفوعة + وثائق + دفع) — الأهم والميزة المميّزة

> **تصحيح مهم (دمج مرحلتين سابقتين):** كانت الخطة تفصل «التسجيل» عن «دعم الدعوات» كمرحلتين متتاليتين. حسب [توثيق-تطبيق-JAZ.md](توثيق-تطبيق-JAZ.md) §5.6: *«دعم الدعوات ليس ميزة منفصلة — هو جزء من التسجيل نفسه. العميل يُكمل التسجيل، ونحن نُصدر الدعوة»*. لذا دُمجتا هنا في مرحلة واحدة: نموذج تسجيل متعدّد الخطوات ينتهي بإصدار الدعوة مباشرة.
>
> JAZ يُصدر **دعوة/بطاقة دخول** فقط — لسنا مكتب تأشيرات ولا نضمن الفيزا. الدعوة تساعد العميل على طلب الفيزا بنفسه من السفارة. التأمين/الموعد/الـApplication خدمات مساندة اختيارية مدفوعة (انظر توثيق §1 و§6).

**الهدف:** تحقيق ميزة «يبدأ بالموقع ويُكمل بالتطبيق»، وينتهي التسجيل بإصدار الدعوة والخدمات المساندة دون خروج من نفس التدفّق.

**الخطوات (توثيق §5.6):**
- [ ] **نموذج تسجيل متعدّد الخطوات** مع مؤشّر تقدّم:
  1. البيانات المهنية (**تعبئة مسبقة** من الملف الشخصي — المرحلة 1)
  2. **اختيار الخدمات المدفوعة**: تذكرة الدخول (سعر ثابت) · تأمين صحي (دولة + تاريخ بدء/انتهاء) · مساعدة Application للسفارة · حجز موعد السفارة/TLS
  3. **رفع الوثائق** عبر الكاميرا (`expo-camera`) إلى Supabase Storage — قائمة وثائق تختلف حسب الفعالية (`registration_config`)
  4. **الدفع** (بوّابة دفع تُحدَّد حسب السوق العراقي/المنطقة — **قرار تجاري يسبق البرمجة**)
- [ ] **الحفظ التلقائي** في `registrations.form_data` بعد كل خطوة (لا يضيع شيء عند الانقطاع)
- [ ] **تسعير التأمين** = متوسّط أسعار شركات التأمين + هامش ربح، يُعرض للعميل كسعر نهائي (**المكان الوحيد الذي يظهر فيه سعر في التطبيق**). *مُوصى به:* البدء بجدول أسعار يدوي لكل دولة في قاعدة البيانات بدل ربط API فوري — أبسط وأأمن (توثيق §11)
- [ ] **تتبّع حالة الطلب:** تقديم ← تدقيق ← قبول/رفض/طلب تعديل ← إصدار الدعوة ← تسهيل السفر — تدار من `dashboard/registrations` و `dashboard/events/[id]`
- [ ] **الوثائق النهائية كملف PDF واحد** بعد القبول والدفع (تأمين + Application + الموعد مجمّعة) قابل للتحميل من «حسابي»
- [ ] **إشعار** عند كل تغيير في الحالة
- [ ] **شعار داخل الموقع**: «أكمل خطواتك من التطبيق» (تعديل بسيط على الموقع)
- [ ] *(يتطلب جداول جديدة:* `invitation_requests` *+ جدول وثائق مرتبط، إضافة لحقول* `registrations` *الحالية)*

**يعتمد على:** المرحلة 2.
**المخرج:** العميل يسجّل، يختار خدماته، يرفع وثائقه، يدفع، ويحصل على دعوته — كل ذلك في تدفّق واحد متصل بالكامل بين الموقع والتطبيق.

ميزة كبيرة: المطوّر **لا يبدأ من الصفر**. كثير من ملفات الموقع تُنقل للتطبيق كما هي أو بتعديل بسيط، وبعضها يفيد في أكثر من مهمة:

### الشعارات والصور (مجلد `public/`)
- **الشعار الرئيسي:** `Joint Annual Zone logo.png` — للأيقونة وشاشة البداية والهيدر.
- **شعارات القطاعات الأربعة (SVG):** `JAZ Medical.svg` · `JAZ Technology.svg` · `JAZ Industrie.svg` · `JAZ Academia.svg` — تُستخدم في الفلترة، بطاقات القطاعات، والتوصيات.
- **خلفيات وصور جاهزة:** `hero-globe-bg.png` · `invitation-hero-bg.png` · `iraq-map.png` · `world-map.svg` · `og-image.png` — تصلح للخلفيات وبطاقة المشاركة.

### الكود المنطقي (مجلد `lib/`) — يُنقل مباشرة
- **`database.types.ts`** — أنواع TypeScript لكل جداول قاعدة البيانات؛ تُنسخ كما هي = توفير وقت كبير وأمان للبيانات.
- **`i18n/translations.ts`** — كل النصوص العربية/الإنجليزية جاهزة؛ لا حاجة لإعادة الترجمة.
- **`chatbot/faq.ts`** — أسئلة وأجوبة المساعد الذكي جاهزة للمرحلة 6.
- **`form-fields.ts`** — تعريفات حقول النماذج؛ يُعاد استخدامها في شاشة التسجيل.
- **`supabase/client.ts`** — طريقة الاتصال بقاعدة البيانات (بتعديل بسيط لبيئة الموبايل).
- **`events-visibility.ts`** + **`utils.ts`** — منطق عرض الفعاليات ودوال مساعدة.

### بيانات جاهزة (نفس قاعدة Supabase)
- آلاف الفعاليات، القطاعات، المقالات، والبيانات الثنائية اللغة — **متاحة فوراً** للتطبيق دون أي إدخال جديد.

> **الخلاصة:** الشعارات والأنواع والترجمات والـ FAQ والبيانات كلها قابلة لإعادة الاستخدام، مما يختصر زمن التطوير بشكل ملحوظ — خاصة في المراحل 1، 2، 3، و6.
