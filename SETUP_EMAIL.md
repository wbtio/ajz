# إعداد نظام البريد الإلكتروني للتسجيلات
http://localhost:3000/dashboard/events
## الخطوات المطلوبة

### 1. إنشاء حساب Resend

1. انتقل إلى [resend.com](https://resend.com)
2. قم بإنشاء حساب جديد أو تسجيل الدخول
3. انتقل إلى [API Keys](https://resend.com/api-keys)
4. قم بإنشاء مفتاح API جديد

### 2. إضافة مفتاح API إلى ملف البيئة

افتح ملف `.env.local` وأضف السطر التالي:

```bash
RESEND_API_KEY=re_your_actual_api_key_here
```

### 3. التحقق من النطاق (اختياري ولكن موصى به)

لإرسال رسائل بريد إلكتروني من نطاقك الخاص:

1. في لوحة تحكم Resend، انتقل إلى [Domains](https://resend.com/domains)
2. أضف نطاقك (مثل: `jaz-iraq.com`)
3. أضف سجلات DNS المطلوبة إلى مزود النطاق الخاص بك
4. انتظر حتى يتم التحقق من النطاق

### 4. تحديث عنوان المرسل (إذا تم التحقق من النطاق)

بعد التحقق من النطاق، قم بتحديث عنوان المرسل في `lib/email.ts`:

```typescript
from: 'JAZ Registration <noreply@jaz-iraq.com>',
```

### 5. اختبار النظام

1. قم بتشغيل التطبيق: `npm run dev`
2. انتقل إلى صفحة أحد القطاعات
3. املأ نموذج التسجيل
4. تحقق من:
   - حفظ البيانات في قاعدة البيانات (لوحة التحكم > تسجيلات القطاعات)
   - استلام بريد إلكتروني على `jaz.registr@gmail.com`
   - استلام بريد تأكيد على البريد الإلكتروني للمستخدم

## الميزات المفعلة

✅ **حفظ البيانات في قاعدة البيانات**
- جميع بيانات النموذج تُحفظ في جدول `sector_registrations`
- يتم تخزين: الاسم، البريد الإلكتروني، الهاتف، وجميع حقول النموذج

✅ **إرسال بريد إلكتروني للإدارة**
- يتم إرسال إشعار إلى `jaz.registr@gmail.com`
- يحتوي على جميع تفاصيل الطلب

✅ **إرسال بريد تأكيد للمستخدم**
- يتم إرسال رسالة تأكيد للمستخدم
- تحتوي على ملخص طلبه ومعلومات التواصل

✅ **عرض البيانات في لوحة التحكم**
- صفحة `/dashboard/sector-registrations` تعرض جميع الطلبات
- يمكن مشاهدة جميع التفاصيل المقدمة

## ملاحظات

- إذا لم يتم تكوين `RESEND_API_KEY`، سيتم حفظ البيانات في قاعدة البيانات فقط دون إرسال رسائل بريد إلكتروني
- تحقق من سجلات الخادم (Console) لمعرفة أي أخطاء في إرسال البريد الإلكتروني
- خطة Resend المجانية تسمح بـ 3000 رسالة بريد إلكتروني شهرياً

## استكشاف الأخطاء

### لا يتم إرسال رسائل البريد الإلكتروني

1. تحقق من أن `RESEND_API_KEY` موجود في `.env.local`
2. تحقق من صحة مفتاح API
3. راجع سجلات الخادم للحصول على تفاصيل الخطأ
4. تأكد من إعادة تشغيل خادم التطوير بعد إضافة المتغير

### رسائل البريد الإلكتروني تذهب إلى البريد المزعج

1. قم بالتحقق من نطاقك في Resend
2. أضف سجلات SPF وDKIM وDMARC
3. استخدم عنوان بريد إلكتروني من نطاق تم التحقق منه
