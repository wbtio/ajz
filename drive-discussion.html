نقاش: ربط Google Drive بلوحة تحكم JAZ

نقاش تقني كامل حول كيفية جعل لوحة تحكم موقع JAZ المكان المركزي لشغل الشركة — مستندات، ملفات، تخزين — بدل التبعثر بين Drive و Hostinger وأماكن ثانية.

Google Drive API Service Account Next.js Supabase Auth
— السؤال الأول
زين انت تعرف Google Drive كيف اربط مع جميع الاجهزة؟ تقصد شلون تربط Google Drive على كل الأجهزة بالمكتب (نفس الحساب أو حسابات موحدة تتشارك بيها الملفات)، أو تقصد شي ثاني — زي ربط Google Drive مع تطبيق/موقع JAZ عن طريق API؟
— الإجابة
نكدر اربط مع لوحة التحكم مالت موقع جاز ويكون بشكل متناسق. تقصد تربط Google Drive بلوحة تحكم موقع JAZ (يعني Next.js / Vercel / Supabase) بحيث الملفات تنرفع أو تنعرض أو تنتزامن من درايف داخل لوحة التحكم؟
Q: شنو بالضبط تريد يصير بلوحة التحكم؟
A: رفع/تخزين ملفات (صور، مستندات) من الموقع مباشرة لـ Google Drive
الفكرة من لوحة تحكم موقع JAZ، أي ملف ينرفع (صورة منتج، مستند، وثيقة) يروح تلقائياً يترزن على Google Drive بدل ما يترزن على Supabase Storage أو سيرفر ثاني.
— السؤال الثاني
زين اريد شغل مالت الشركة البسيط كلها يكون في لوحة التحكم الموقع.
— الإجابة
فهمت — يعني مو بس رفع ملفات، تريد لوحة التحكم مال الموقع تصير المكان المركزي لعموم شغل الشركة البسيط (documents، ملفات، ربما مهام بسيطة)، بدل ما يكون مبعثر بين Drive و Hostinger وأماكن ثانية.
المشكلة الحالية الملفات مبعثرة: صور بالـ Hostinger، مستندات بـ Drive شخصي، ملفات بـ Supabase. الموظف ما يدري وين يلاقي شي، ولوحة التحكم تعرض بس الفعاليات والتسجيلات — مو شغل الشركة اليومي.
Q: لوحة التحكم تريدها تصير شنو بالضبط؟
A: واجهة تعرض ملفات Drive (التخزين يظل بـ Drive)
القرار الأذكى ما تكرر التخزين — بس تسوي واجهة (viewer) داخل لوحة تحكم JAZ تعرض محتويات Google Drive بشكل منظم. التخزين يظل بـ Drive (مجاني 15GB)، والواجهة تعرض وتنظّم.

1. المصادقة (Authentication)
الخيار الوصف الأناسب لـ JAZ
Service Account — موصى به حساب خدمة مو حساب شخصي. يوصل لمجلد Drive محدد بدون تسجيل دخول يدوي. للشركة الصغيرة — أبسط وأنسب
OAuth لكل مستخدم — بديل كل موظف يسجل دخول بحسابه Google الشخصي. أعقد، تحتاجه بس لو تريد كل موظف يشوف ملفاته الشخصية. مبالغ فيه لشركة صغيرة
2. الـ API Route (Next.js backend)
تسوي endpoint مثل /api/drive/files يستخدم مكتبة googleapis (الرسمية من Google) يسوي:
  • drive.files.list() — يرجع قائمة الملفات
  • كل ملف يرجع: اسم، نوع، تاريخ تعديل، رابط، thumbnail لو صورة
  • كلها من المجلد المحدد
3. الواجهة (Frontend)
صفحة بلوحة التحكم تعرض:
  • شبكة/قائمة بالملفات (زي متصفح ملفات بسيط)
  • فلترة حسب النوع (صور / مستندات) أو المجلد
  • زر "فتح بـ Drive" لكل ملف (يفتح تاب جديد بجوجل درايف نفسه للتعديل)
  • ممكن تضيف بحث بالاسم
4. الصلاحيات
تربطها مع نظام تسجيل الدخول الحالي بلوحة التحكم (Supabase Auth) — يعني بس الموظفين المسجلين يشوفون الملفات، مو أي زائر.
— خطوات التنفيذ العملية
تفعيل Google Drive API + إنشاء Service Account
ادخل Google Cloud Console ← إنشاء مشروع جديد ← فعّل Google Drive API ← IAM & Admin ← Service Accounts ← إنشاء حساب ← نزّل ملف JSON فيه المفاتيح.
مشاركة المجلد مع Service Account
أنشئ مجلد بـ Google Drive (مثلاً "JAZ Uploads") ← شاركه مع إيميل الـ Service Account (ينتهي بـ .iam.gserviceaccount.com) ← أعطه صلاحية Editor.
تثبيت googleapis بالمشروع

بناء الـ API Route
ملف app/api/drive/files/route.ts — يستعمل Service Account للوصول للدرايف ويرجع قائمة الملفات.
بناء صفحة العرض بالـ Frontend
صفحة app/dashboard/files/page.tsx — تعرض الملفات بشبكة أو قائمة، مع فلترة وزر "فتح بـ Drive".
— كود الـ API Route المقترح
هذا الكود الأساسي لـ app/api/drive/files/route.ts:
import { NextResponse } from "next/server"; import { google } from "googleapis"; export const runtime = "nodejs"; const auth = new google.auth.GoogleAuth({ keyFile: process.env.GOOGLE_SERVICE_ACCOUNT_PATH, scopes: ["https://www.googleapis.com/auth/drive.readonly"], }); export async function GET() { const drive = google.drive({ version: "v3", auth }); const res = await drive.files.list({ q: "'FOLDER_ID' in parents and trashed = false", fields: "files(id, name, mimeType, modifiedTime, thumbnailLink, webViewLink)", orderBy: "modifiedTime desc", pageSize: 50, }); return NextResponse.json({ files: res.data.files }); }
متغيرات البيئة المطلوبة GOOGLE_SERVICE_ACCOUNT_PATH — مسار ملف JSON اللي نزلته من Google Cloud.
GOOGLE_DRIVE_FOLDER_ID — ID المجلد (تلاقيه بالـ URL بعد تفتح المجلد بـ Drive).
— كود صفحة العرض (مختصر)
// app/dashboard/files/page.tsx "use client"; import { useState, useEffect } from "react"; interface File { id: string; name: string; mimeType: string; modifiedTime: string; thumbnailLink?: string; webViewLink?: string; } export default function FilesPage() { const [files, setFiles] = useState<File[]>([]); useEffect(() => { fetch("/api/drive/files") .then(r => r.json()) .then(data => setFiles(data.files)); }, []); return ( <div className="grid grid-cols-4 gap-4"> {files.map(f => ( <div key={f.id} className="rounded-xl border p-4"> {f.thumbnailLink && <img src={f.thumbnailLink} />} <p>{f.name}</p> <a href={f.webViewLink}>فتح بـ Drive</a> </div> ))} </div> ); }
— نقاط تحتاج قرار قبل التنفيذ
نقطة 1 — رفع أم عرض؟ هل تريد بس عرض الملفات الموجودة بالـ Drive؟ أو تريد أيضًا رفع ملفات جديدة من لوحة التحكم مباشرة؟ الرفع يحتاج صلاحية drive.file بدل drive.readonly.
نقطة 2 — مجلد واحد أم متعدد؟ هل كل الملفات بمجلد واحد "JAZ Uploads"؟ أو تريد مجلدات فرعية منظّمة (صور الفعاليات / مستندات الشراكات / عقود / فواتير)؟ لو متعددة، نضيف قائمة مجلدات باليسار زي متصفح ملفات.
نقطة 3 — صلاحيات الأدوار هل كل الموظفين يشوفون كل الملفات؟ أو المدير يشوف كل شي، والموظف العادي يشوف مجلداته فقط؟ هذا يربط بنظام الأدوار الموجود (admin / editor / viewer).
نقطة 4 — البحث هل تريد بحث بالاسم فقط (سهل)? أو بحث داخل المحتوى (يحتاج Google Drive Search API — أعمق لكن أبطأ)؟
نقطة 5 — التكلفة Google Drive API مجاني حتى 1,000,000 طلب باليوم. لشركة صغيرة، هذا أكثر من كافي. لا توجد كلفة إضافية.

الأنسب لـ JAZ مستندات الشركة صور الفعاليات
التوصية استعمل الاثنين معًا: Google Drive للمستندات (عقود، فواتير، تقارير — لأنها قابلة للتعديل والمشاركة)، و Supabase Storage للصور (صور الفعاليات، شعارات الشركاء — لأنها سريعة و مدمجة). لوحة التحكم تعرض كلاهما في صفحة واحدة منظّمة.
— السؤال الأخير قبل التنفيذ
تريد أبدأ أبني لك الكود (الـ API route + صفحة العرض) هسه، لو تفضل نمشي خطوة بخطوة وتسوي أنت الإعداد بـ Google Cloud أول (Service Account) وبعدين نرجع للكود؟
الخطوة الأولى عمليًا إذا جاهز، ادخل console.cloud.google.com وأنشئ المشروع + Service Account. نزّل ملف JSON وحطه بمجلد المشروع (خارج public!). بعدين أبنيلك الكود كامل.
